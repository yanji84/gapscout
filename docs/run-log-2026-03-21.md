## Round 1 → Round 2 Improvements Applied

### Code Changes (pain-points.mjs)

**Fix 1: Sentiment context awareness**
- Added `sentimentMultiplier()` — checks 80-char window around keyword for positive/negative context
- Positive context (happy, love, great, worth it) → 0.2x discount
- Negative context (hate, regret, waste, terrible) → 1.5x boost
- Added `matchSignalsWeighted()` for use in scoring

**Fix 2: Non-pain post penalties**
- Added `NON_PAIN_TITLE_SIGNALS`: tips, workflow, share your, guide, showcase, tutorial, etc.
- Added `NON_PAIN_FLAIRS`: appreciation, resources, tips, showcase, etc.
- Title match → -4.0 penalty, flair match → -2.0 penalty

**Fix 3: Broader agreement detection**
- Added `EXPANDED_AGREEMENT` keywords: "absolutely", "this exactly", "+1", "preach", "for real", "100%", etc.
- Added thematic agreement: comments sharing same pain category as post
- Upvote-weighted agreement: high-score comments count more
- Combined agreement = direct + (thematic * 0.5)
- Lowered validation thresholds to match broader input

**Additional fixes:**
- WTP split: strong (intent, 3.0 weight) vs weak (mentions, 0.5 weight)
- Timestamp probe: queries PullPush for latest data timestamp
- TOOL_NOISE expanded: AI, API, LLM, MVC, Honestly, Actually, etc.

## Round 2 → Round 3 Improvements Applied

### Code Changes (pain-points.mjs)

**R2-Fix 1: Engagement cap + zero-pain penalty**
- Engagement contribution capped at 4.0 total (was unlimited, could reach 12+)
- Posts with zero pain signals AND zero WTP signals get -5.0 penalty
- This is the single highest-impact change

**R2-Fix 2: Non-pain filter improvements**
- Added standalone "guide" (not just "guide to"), "how to", "strategy", "climbing to", "deck list", "tier list", "where did everyone"
- Flair matching changed from exact `has()` to substring `some(f => flair.includes(f))`
- Added domain-relevant non-pain flairs: "deck", "pulls", "meme", "humor", "art", "collection"

**R2-Fix 3: Intensity gated on pain signals**
- If `painWeight === 0`, intensity boost is zeroed out
- Prevents strategy guides from getting +4.5 from "absolutely", "completely", "constantly"

## Remaining Known Issues

1. **No domain-specific scan queries**: Tool uses generic frustration keywords; misses scalping, pull rates, pack fraud, grading complaints, set rotation pain
2. **"Looking for" is noisy**: Matches any seeking behavior, not just pain-driven seeking
3. **WTP keywords are domain-blind**: "bought" in TCG = normal behavior, not willingness to pay for a solution
4. **Agreement detection still fires on non-pain threads**: Strategy agreement counted same as frustration agreement
5. **No discover phase filtering**: Coupon/deal/meme subreddits still need blocklist

## Suggested Next Steps

1. Add configurable domain-specific scan queries (`--scan-queries` flag or built-in domain profiles)
2. Gate agreement detection on `postPainCategories.length > 0`
3. Add subreddit blocklist patterns to discover phase
4. Consider requiring at least 1 pain signal for a post to appear in results at all
